# Contributing to the Study GRC Website

Thank you for helping us improve the Study GRC website! As a community built and community-driven non-profit, we rely on collaboration to keep our platform helpful, accessible, and current for everyone.

💬 **Need help or want to discuss an idea?** Join the conversation in our [Study GRC Discord Server](https://discord.studygrc.org).

---

## Ways to Contribute

You do not need to be a web developer to help. Choose the path that works best for you:

- **Open a ticket:** If you spot a broken link, a typo, or a design issue, go to the **Issues** tab and open a ticket describing what you found.
- **Suggest a feature:** Have an idea for a new section? Open a ticket to discuss it with the community before writing any code.
- **Submit a change:** If you are ready to edit the HTML, CSS, or JavaScript yourself, you can open a Pull Request.

## How to Submit Changes (Pull Requests)

We use the standard GitHub workflow to review and approve changes together. Here is a clear guide to submitting your updates:

1. **Fork the repository:** Click the "Fork" button at the top right of the page to create your own workspace.
2. **Create a branch:** Name your branch specifically for your task (for example, `fix-mobile-menu` or `update-speaker-list`).
3. **Make your changes:** Edit the HTML, CSS, or JS files. Since this is a static site, you can just save your files and refresh your web browser to test your changes. 
4. **Commit your changes:** Save your work with a clear message explaining your updates (for example, "Fixed broken link in the footer").
5. **Push your branch:** Upload your saved changes to your GitHub account.
6. **Open a Pull Request:** Return to the main Study GRC Website repository and click "New Pull Request" to ask the community to review and merge your work.

## Guidelines for Website Updates

To make sure the website remains fast, accessible, and easy for anyone to maintain, please follow these simple practices:

- **Keep it simple:** We use plain HTML, CSS, and JavaScript. Please ask in a ticket before adding new frameworks or complex libraries.
- **Use plain language:** Write clear, concise text that is easy for a broad audience to understand.
- **Prioritize accessibility:** Make the site usable for everyone. Use standard HTML tags, add descriptive `alt` text to images, and check that colors are easy to read.
- **Test on mobile devices:** Our community uses many different devices. Resize your browser window to verify your changes work well on smaller screens.
